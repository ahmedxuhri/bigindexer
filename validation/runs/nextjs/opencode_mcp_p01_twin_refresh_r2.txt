## Project Overview

**VERIFIED** — This is the [Next.js](https://nextjs.org) framework monorepo (`packages/next/src/` published as the `next` npm package). It's a React framework providing server-side rendering, file-based routing (App Router + Pages Router), static generation, API routes, and middleware. ## Strong Points

| Claim | Status | Evidence |
|---|---|---|
| **Dual-bundler strategy (Turbopack + webpack)** gives performance + compatibility safety net | **VERIFIED** | AGENTS.md: "Turbopack is the default bundler for both next dev and next build. To force webpack: next build --webpack". This means fast Rust-based dev iteration without abandoning webpack's ecosystem. |
| **Monorepo isolation** with well-separated packages (`next`, `create-next-app`, `eslint-plugin-next`, `font`, `third-parties`, `next-swc`) | **VERIFIED** | AGENTS.md lines 7-26: clear directory structure with separate publishable packages. |
| **Compiled React** is the central coupling hub — all build, client, SWC, and CLI systems depend on it | **VERIFIED** | Architecture summary: `compiled_react` cluster has 107k+ edges to webpack build, 38k+ to client components, 20k+ to SWC. This is a **highly leveraged** dependency management strategy — vendoring React gives full control over the framework-React interface. |
| **Rich test infrastructure** with per-mode commands (`test-dev-turbo`, `test-start-webpack`, `test-unit`, `testheadless`) and `NEXT_SKIP_ISOLATE` for fast iteration | **VERIFIED** | AGENTS.md testing section: "NEXT_SKIP_ISOLATE=1 - skip packing Next.js for each test (~100s faster)". |
| **Turbopack + Rust crates** for SWC transforms give native-speed builds | **VERIFIED** | AGENTS.md: "crates/ - Rust crates for Next.js SWC bindings", "turbopack/ - Turbopack bundler (Rust)". |

## Weak Points

| Claim | Status | Evidence |
|---|---|---|
