# Pydantic Project Analysis (Evidence Mode)

## What This Project Is

**VERIFIED** — Pydantic is a data validation library using Python type hints.  
Source: `README.md:13` — *"Data validation using Python type hints."*

**VERIFIED** — It has a **hybrid Python + Rust architecture**: Python layer (~45.5k LOC in 104 files) generates `CoreSchema` dicts from Python type annotations, then passes them to the Rust `pydantic-core` (~33k LOC) for execution (validation/serialization).  
Sources: `pyproject.toml:49` (pinned `pydantic-core==2.46.0`), `pydantic/_internal/_generate_schema.py:2932` (single core bridging module), `pydantic/main.py:30-33` (imports from `pydantic_core`).

**VERIFIED** — Minimum Python 3.10+, MIT licensed, originally by Samuel Colvin. Current stable release v2.13.3 (April 2026).  
Sources: `pyproject.toml:22,44`, `HISTORY.md:5`.

**VERIFIED** — The test suite is 68,947 lines across 167 files (1.5× the product code by volume).  
Source: `find tests/ -name '*.py' | wc -l` = 167, `cat | wc -l` = 68947.

## Weak Points

**VERIFIED**

| Weakness | Evidence |
|---|---|
| **Lockstep release coupling** — `pydantic-core` dependency is pinned to an *exact version* (`==2.46.0`). Both must be released simultaneously. Any change to pydantic-core requires a pydantic release. | `pyproject.toml:49` — `'pydantic-core==2.46.0'` |
| **`_generate_schema.py` is a monolith** — 2,932 lines handling all type-to-schema conversion. High complexity, difficult to navigate, single point of change for most type-related work. | `wc -l pydantic/_internal/_generate_schema.py` = 2932 |
| **Circular import fragility** — Extensive use of `TYPE_CHECKING` guards, `_import_utils.py` (20 lines of lazy cached imports), and `__getattr__`-based lazy loading suggests import ordering is brittle. | `pydantic/_internal/_import_utils.py`, `pydantic/__init__.py` pattern of `TYPE_CHECKING`-only imports via `__getattr__` |

## Historical Context

**HYPOTHESIS** — Pydantic v2 (current, 2023–present) was a ground-up rewrite from v1, motivated by performance. The architecture shifted from pure Python (v1) to Rust core + Python schema generation (v2).  
Source: `README.md:25` — *"Pydantic V2 is a ground-up rewrite that offers many new features, performance improvements"*
