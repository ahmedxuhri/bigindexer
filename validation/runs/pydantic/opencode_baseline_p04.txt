## Historical Context

V1 was vendored at pydantic v2 launch as `pydantic.v1` to ease migration. It is intentionally kept minimal/stable — no new features, no structural changes. The v2 plugin system was added later but was never backported to v1.

## Safest Implementation Path

Given **zero existing observability, no plugin system, and a mandate for frozen stability**, the safest path is:

### Strategy: Module-level callback registry + opt-in import hook

**Do NOT touch** `ModelMetaclass`, `BaseModel` class body, or `validate_model` signatures. Instead, add at the **bottom** of `pydantic/v1/main.py` (after line 1130) a minimal hook point:

```
pydantic/v1/main.py
  ┌──────────────────────────────────────────────┐
  │  (existing 1130 lines - untouched)           │
  ├──────────────────────────────────────────────┤
  │  # NEW: opt-in observer hooks (lines 1131+) │
  │  _OBSERVER = None                            │
  │                                              │
  │  def _set_observer(observer): ...            │
  │                                              │
  │  # wrap only 3 entry points:                 │
  │  #   BaseModel.__init__                      │
  │  #   BaseModel.dict / .json                  │
  │  #   validate_model()                        │
  └──────────────────────────────────────────────┘
```

### Key design decisions:

| Concern | Decision | Rationale |
|---------|----------|-----------|
| **Dependencies** | `None` — standard library only | Zero new imports. Observer is a duck-typed callback. |
| **Activation** | Import-time opt-in via `_set_observer()` | No env vars, no global state mutation on import. Must be explicitly installed. |
| **Boundary** | Strictly within `pydantic/v1/main.py` | No changes to `__init__.py`, other v1 modules, or any v2 code. |
| **Performance** | `if _OBSERVER is not None` guard | Single pointer comparison at each hook point; zero overhead when disabled. |
| **Wrapping targets** | `BaseModel.__init__` (line 355), `validate_model` (line 1053), `BaseModel.dict` (line 450), `BaseModel.json` (line 484) | These are the 4 public choke points. Do NOT wrap inner loops (`_iter`, `_get_value`, field validation). |

### Exact insertion points (with no behavioral change):

1. **`validate_model()`** (line 1053) — wrap the entire function body so that the caller gets `on_enter`/`on_success`/`on_error` around the full validation pipeline
2. **`BaseModel.__init__`** (line 355) — wrap the constructor call to `validate_model()`
3. **`BaseModel.dict()`** (line 450) and **`BaseModel.json()`** (line 484) — wrap serialization

### Observer interface (duck-typed, no abstract base):

```python
class V1Observer:
    def on_model_init_enter(self, model_class, data): ...
    def on_model_init_success(self, model_class, data, instance): ...
    def on_model_init_error(self, model_class, data, error): ...

    def on_validate_enter(self, model_class, input_data): ...
    def on_validate_success(self, model_class, input_data, values): ...
    def on_validate_error(self, model_class, input_data, error): ...

    def on_serialize_enter(self, instance, method): ...  # method='dict'|'json'
    def on_serialize_success(self, instance, method, result): ...
    def on_serialize_error(self, instance, method, error): ...
```

### What to avoid:

- **Do NOT** add `__pydantic_init_subclass__` or `__init_subclass__` — would break existing v1 subclasses
- **Do NOT** modify `ModelMetaclass.__new__` — too risky, called at class creation time
- **Do NOT** add module-level `import logging` or other dependencies — violates zero-cost principle
- **Do NOT** wrap `_iter`, `_get_value`, field validators — too hot-path, would cause performance regression
- **Do NOT** modify any file outside `pydantic/v1/main.py` — cross-boundary impact
- **Do NOT** touch `pydantic/v1/__init__.py` — changing re-export behavior would affect all v1 consumers
