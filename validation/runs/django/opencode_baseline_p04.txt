
Key boundaries crossed during URL resolution:
1. **Module import boundary** — `URLResolver.urlconf_module` imports the dotted path
2. **Cache boundary** — `_get_cached_resolver` is process-wide `@functools.cache`
3. **Thread-local boundary** — `set_urlconf()` / `get_urlconf()` via `asgiref.local.Local`
4. **Test isolation boundary** — `@override_settings` patches `ROOT_URLCONF` per test class

#### Recommended Path: Module-level Logging Hook (0 cross-boundary crossings)

Leverage the fact that Python modules execute at import time. Add structured observability *inside the URLconf module itself*, with zero changes to framework code, middleware, or test runner:

**Blueprint** (`tests/generic_inline_admin/urls.py`):

```python
import logging
from django.urls import path
from . import admin

logger = logging.getLogger("django.request")  # reuse existing logger

urlpatterns = [
    path("generic_inline_admin/admin/", admin.site.urls),
]

# Structured diagnostics at import time — zero cross-boundary impact
logger.log(
    logging.DEBUG,
    "URLconf loaded: %s — %d pattern(s): %s",
    __name__,
    len(urlpatterns),
    [p.pattern.describe() if hasattr(p, 'pattern') else str(p) for p in urlpatterns],
)
```

**Why this is safest**:
- **Zero framework changes** — no middleware, no monkey-patching, no signal hooks
- **Zero test infrastructure changes** — no new settings, no runner modifications
- **Zero cross-boundary impact** — stays entirely within the URLconf module's own import boundary
- **Reuses existing `django.request` logger** — consistent with `log.py:11` and `runtests.py:235-239` (though suppressed at ERROR level by default; enabling DEBUG logging would require only a test-runner config change)
- **No caching interference** — does not touch `_get_cached_resolver` cache or thread-local state
- **No test mutation** — `@override_settings` isolation is preserved

#### Alternative: Resolution-time Wrapper (1 boundary crossing: URLPattern.func)

If import-time diagnostics are insufficient and you need *per-request* observability without middleware:

```python
import logging
from django.urls import path
from functools import wraps
from . import admin

logger = logging.getLogger("django.request")

original_urlpatterns = [
    path("generic_inline_admin/admin/", admin.site.urls),
]

def _observe_resolve(view_func):
    @wraps(view_func)
    def wrapper(request, *args, **kwargs):
        logger.debug("Resolved: %s → %s.%s", request.path_info,
                     view_func.__module__, view_func.__name__)
        return view_func(request, *args, **kwargs)
    return wrapper

# Wrap each pattern's callback — crosses URLPattern.func boundary only
urlpatterns = []
for p in original_urlpatterns:
    p.callback = _observe_resolve(p.callback)
    urlpatterns.append(p)
```

**Trade-off**: Crosses one boundary (overwrites `URLPattern.callback`), but with no impact on caching, middleware, settings, or test isolation.
