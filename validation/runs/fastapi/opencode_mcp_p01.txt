## Evidence Mode Analysis: FastAPI (v0.136.1)

### What This Project Is

**FastAPI** is a Python web framework for building APIs using standard type hints, built atop **Starlette** (ASGI) and **Pydantic** (validation).

- **VERIFIED** — `fastapi/applications.py:41`: `class FastAPI(Starlette)` — inherits from Starlette directly
- **VERIFIED** — `pyproject.toml:44-50`: runtime deps are `starlette>=0.46.0` + `pydantic>=2.9.0`
- **VERIFIED** — `README.md:30-41`: explicitly states "high performance", "standards-based (OpenAPI + JSON Schema)", "automatic interactive docs"
- **VERIFIED** — `fastapi/__init__.py` exists, version `0.136.1`

### Strong Points

**1. Type-hint-driven API declaration is genuinely innovative**
- **VERIFIED** — `fastapi/param_functions.py` — full public API using `Annotated[int, Path()]` pattern; `fastapi/params.py:745` — `Depends` dataclass for dependency injection
- **VERIFIED** — `fastapi/dependencies/utils.py` — `get_dependant()` (line ~50) introspects callable signatures to build a dependency tree; `solve_dependencies()` resolves it at runtime
- **HYPOTHESIS** — This approach is unique among Python web frameworks at this scale; no other major framework uses Python type hints as the primary DSL for request declaration (Flask, Django all use string-based or class-based approaches). Reasonable conclusion from comparing with Flask/Django patterns, but not proven by this repo alone.

**2. Automatic OpenAPI generation is comprehensive and correct**

### Weak Points

**1. Monolithic core files (maintainability concern)**
- **VERIFIED** — `fastapi/applications.py:4691` lines — the `FastAPI` class is extremely long
- **VERIFIED** — `fastapi/routing.py:4956` lines — the router is nearly 5000 lines
- **HYPOTHESIS** — Files of this size increase cognitive load for new contributors and make merge conflicts more likely. This is a code organization weakness rather than a functionality bug. The 100% coverage requirement mitigates but doesn't eliminate this concern.

**2. Deprecated code still shipped in the package**
- **VERIFIED** — `fastapi/responses.py:39-97` — `UJSONResponse` and `ORJSONResponse` emit `FastAPIDeprecationWarning` but still exist as ~60 lines of dead weight
- **VERIFIED** — same pattern for `regex`→`pattern`, `example`→`examples`, `extra`→`json_schema_extra` — referenced in deprecation infrastructure
- **HYPOTHESIS** — While reasonable for a mature framework (breaking changes are painful for users), carrying deprecated code indefinitely adds surface area and cognitive overhead. This is a tradeoff, not a bug.

**3. Heavy dependency tree even for basic use**
- **VERIFIED** — `pyproject.toml:44-50` — core deps: starlette, pydantic, typing-extensions, typing-inspection, annotated-doc
- **VERIFIED** — `pyproject.toml:59-77` — `standard` extras pull in httpx, jinja2, python-multipart, uvicorn, pydantic-settings, pydantic-extra-types, fastapi-cli, fastar
- **HYPOTHESIS** — If you only need a simple JSON API, the minimal install still requires Starlette + Pydantic + typing stack, which is heavier than, say, a Flask single-file app. This is inherent to FastAPI's design philosophy and not a bug.

**4. CLI lives in a separate package, creating a split**
- **VERIFIED** — `fastapi/cli.py:13` lines total — just imports from `fastapi_cli.cli`
- **VERIFIED** — `pyproject.toml:61` — depends on `fastapi-cli[standard] >=0.0.8`
- **HYPOTHESIS** — This means versioning mismatches can occur between `fastapi` and `fastapi-cli`. The `standard-no-fastapi-cloud-cli` variant (line 79-95) adds further complexity to the install matrix. Minor concern.

**5. `fastapi-slim` directory exists but is deprecated**
- **VERIFIED** — top-level `fastapi-slim/` directory exists
- **HYPOTHESIS** — Based on README note ("Do not install"), this is a legacy alias. Its presence in the repo is dead weight and could confuse new users. Not verified by reading the fastapi-slim source itself.

**6. Commercial tie-in (FastAPI Cloud) embedded in packaging**
- **VERIFIED** — `pyproject.toml:62` — `standard` group includes `fastapi-cli[standard]` which bundles the cloud CLI
- **VERIFIED** — `pyproject.toml:79-95` — `standard-no-fastapi-cloud-cli` was explicitly created as a workaround
- **VERIFIED** — `README.md:451` — mentions "FastAPI Cloud"
- **HYPOTHESIS** — Bundling a cloud deployment CLI in the default install is a pattern that some open-source consumers may view negatively, especially given the need to create an alternative install target to avoid it.

**7. `Annotated`-required API may be verbose for simple cases**
- **VERIFIED** — `fastapi/param_functions.py` — all parameter functions use `Annotated[T, ...]` pattern
- **HYPOTHESIS** — Simple endpoints like `def get(item_id: int)` now need `Annotated[int, Path()]` for path params, which is more verbose than Flask's `@app.route('/<int:id>')`. This is a deliberate design choice trading verbosity for explicitness and type safety, but it's a real ergonomic cost.
