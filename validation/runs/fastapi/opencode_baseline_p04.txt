## Safest Implementation Path

**Zero codebase modification. User-space only. Use `@app.middleware("http")`.**

```python
import time
from fastapi import FastAPI, Request

app = FastAPI()

@app.middleware("http")
async def add_timing_header(request: Request, call_next):
    start = time.perf_counter()
    response = await call_next(request)
    response.headers["X-Process-Time"] = f"{time.perf_counter() - start:.6f}"
    return response
```

**Why safest (VERIFIED):**
- **Zero cross-boundary impact** — no changes to `build_middleware_stack`, routing, or any internal module. User-space code only.
- **Already tested** — `test_middleware/test_tutorial001.py` validates the exact pattern.
- **Existing pattern** — same code as `applications.py:4628-4636` docstring and `docs_src/middleware/tutorial001_py310.py`.
- **Responds to all routes uniformly** — sits inside ExceptionMiddleware, so even routes that raise HTTPExceptions are timed.

**One caveat (VERIFIED):** For `StreamingResponse` routes, `X-Process-Time` measures time-to-first-byte + serialization, not total transfer. The response object is created but the stream hasn't been consumed when timing ends. If you need full stream timing, wrap `send` in a raw ASGI middleware (moderate boundary impact — must be added via `self.user_middleware`).
